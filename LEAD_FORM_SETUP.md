# 购买意向表单设置说明

这份说明只解决一件事：在还没有正式支付链接前，先被动收集购买意向。

## 推荐表单字段

创建一个普通表单，字段建议如下：

1. 邮箱或微信
2. 想购买什么
3. 你是个人、团队、工作室还是公司
4. 主要用途
5. 预算或意向版本

## 选项建议

“想购买什么”可以设置为：

- ¥99 标题工厂付费模板包
- ¥999 商业授权版
- Pro 订阅版
- 先看样品

“你是个人、团队、工作室还是公司”可以设置为：

- 个人创作者
- 内容团队
- 代运营团队
- 工作室
- 企业运营

## 拿到表单链接后

打开根目录的 `checkout-config.json`，把表单链接填到：

```json
{
  "purchaseIntentUrl": "https://你的表单链接"
}
```

如果同时拿到了真实购买链接，也一起填写：

```json
{
  "templatePackUrl": "https://模板包购买链接",
  "proCheckoutUrl": "",
  "businessLicenseUrl": "https://商业授权购买链接",
  "purchaseIntentUrl": "https://购买意向表单链接"
}
```

## 页面会发生什么

配置 `purchaseIntentUrl` 后，首页的“保存购买意向”会变成“提交购买意向”。用户输入邮箱后，会打开这个表单链接，并自动带上邮箱、来源和页面地址参数。

没有配置时，页面会继续本机保存，不影响用户下载样品包或查看付费页。
